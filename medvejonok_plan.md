## 1. Общая концепция

**Мишка знает** — русскоязычный ИИ-репетитор для школьников 5–11 классов. Работает в формате чата: ученик задаёт вопросы, решает задачи и получает объяснения от YandexGPT. Мишка терпелив, не осуждает и объясняет столько раз, сколько нужно.

**Ключевые отличия от аналогов:**
- Ориентирован на школьную программу РФ (5–11 класс, предметы ФГОС)
- Использует YandexGPT — российский ИИ без зависимости от западных сервисов
- Встроенная математическая клавиатура для ввода формул
- Два режима: свободный чат и режим заданий с проверкой
- Стриминг ответов (текст появляется постепенно, как в ChatGPT)
- Минималистичный дизайн на HeroUI, похожий на современные AI-чаты

**Аналоги:**
| Сервис | Язык | Проблема |
|---|---|---|
| Winny Tutor (winnytutor.com) | EN | Нет русского, нет формул |
| Яндекс Репетитор (yandex.ru/education) | RU | Закрытый, не для школьников |
| Khanmigo (khanacademy.org) | EN | GPT-4, не адаптирован под РФ |

## 2. Технологический стек

### Frontend
- **Next.js 14** (App Router, SSR) — единый проект для фронта и бэкенда
- **HeroUI** — UI-библиотека (минималистичные компоненты)
- **Tailwind CSS** — стилизация
- **KaTeX** — рендеринг математических формул в сообщениях
- **react-markdown** + **remark-math** + **rehype-katex** — парсинг и отображение LaTeX в ответах ИИ

### Backend (Next.js API Routes)
- **Next.js API Routes** (`/app/api/...`) — серверная логика внутри того же проекта
- **better-sqlite3** — SQLite для локального хранения данных
- **Drizzle ORM** — типизированная работа с БД
- **bcrypt** — хэширование паролей
- **jose** — JWT-токены для сессий (без сторонних auth-сервисов)

### ИИ
- **YandexGPT API** (через Yandex Cloud) — основная модель
- **Streaming** через `ReadableStream` / Server-Sent Events (SSE)
- Системный промпт адаптируется под предмет, класс и режим

### Математическая клавиатура
- Кастомная панель на HeroUI `Tabs` + `Button`
- Вставляет LaTeX-фрагменты в поле ввода
- Вкладки: **Основные** (+−×÷), **Функции** (sin cos tan...), **Дроби/степени** (x² √x x/y), **Специальные** (∫ ∑ ∞ → ∈)

### База данных
- **SQLite** (локальный файл `database.db`)
- ORM: **Drizzle** (миграции, типизация)

## 3. Структура базы данных

```sql
-- Пользователи
users (
  id          INTEGER PRIMARY KEY,
  email       TEXT UNIQUE NOT NULL,
  password    TEXT NOT NULL,         -- bcrypt hash
  name        TEXT NOT NULL,
  grade       INTEGER NOT NULL,      -- 5..11
  avatar      TEXT DEFAULT 'bear1',  -- выбор аватара
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
)

-- Сессии чата
chat_sessions (
  id          INTEGER PRIMARY KEY,
  user_id     INTEGER REFERENCES users(id),
  subject     TEXT NOT NULL,         -- math, physics, russian
  title       TEXT,                  -- авто-заголовок из первого сообщения
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
)

-- Сообщения в чате
messages (
  id          INTEGER PRIMARY KEY,
  session_id  INTEGER REFERENCES chat_sessions(id),
  role        TEXT NOT NULL,         -- 'user' | 'assistant'
  content     TEXT NOT NULL,         -- текст (может содержать LaTeX)
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
)

-- Задания (режим практики)
task_sessions (
  id            INTEGER PRIMARY KEY,
  user_id       INTEGER REFERENCES users(id),
  subject       TEXT NOT NULL,
  topic         TEXT NOT NULL,       -- тема задания
  task_text     TEXT NOT NULL,       -- текст задачи от ИИ
  correct       BOOLEAN,             -- NULL пока не проверено
  user_answer   TEXT,
  ai_feedback   TEXT,                -- объяснение ошибки / похвала
  created_at    DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

## 5. Страницы и экраны

### 5.1 Авторизация
- **Регистрация** (`/register`): имя, email, пароль, выбор класса (5–11), выбор аватара из набора
- **Вход** (`/login`): email + пароль
- Сессия хранится в httpOnly cookie (JWT, 7 дней)
- Все роуты `/app/*` защищены middleware

### 5.2 Дашборд (`/dashboard`)
- Приветствие от Мишки с именем ученика
- Карточки предметов для быстрого старта чата
- Последние 3 сессии чата
- Статистика: решено задач сегодня / всего

### 5.3 Чат (`/chat/[sessionId]`)
- **Дизайн**: колонка с историей сообщений + поле ввода внизу (как ChatGPT / Claude)
- **Сайдбар**: список сессий по предметам
- **Сообщения пользователя**: справа, цветной фон
- **Сообщения Мишки**: слева, аватар мишки, белый фон
- **LaTeX**: рендерится через KaTeX прямо в тексте (`$формула$` или `$$блок$$`)
- **Стриминг**: текст появляется по мере генерации (SSE)
- **Математическая клавиатура**: кнопка `∑` в поле ввода → раскрывается панель снизу

### 5.4 Режим заданий (`/tasks`)
1. Ученик выбирает **предмет** → **класс** (берётся из профиля) → **тему**
2. Нажимает «Получить задание» → API генерирует задачу через YandexGPT
3. Задача отображается в карточке (с LaTeX если нужно)
4. Ученик вводит ответ (текст или формула через клавиатуру)
5. Нажимает «Проверить» → API отправляет задачу + ответ в YandexGPT
6. Мишка: ✅ «Верно! Молодец!» или ❌ «Не совсем... вот как правильно:»
7. При ошибке — пошаговое объяснение решения

## 6. Предметы и темы

### Математика (5–11 класс)
- 5–6 кл: дроби, проценты, уравнения, площади фигур
- 7–9 кл: алгебра, системы уравнений, геометрия, функции
- 10–11 кл: производная, интеграл, тригонометрия, вероятность

### Физика (7–11 класс)
- Механика (сила, скорость, ускорение)
- Электричество (закон Ома, сопротивление)
- Термодинамика, оптика, ядерная физика

### Русский язык (5–11 класс)
- Орфография (правописание приставок, суффиксов)
- Пунктуация (запятые, двоеточие, тире)
- Морфология, синтаксис

## 7. YandexGPT: системные промпты

### Базовый системный промпт
```
Ты — Мишка, добрый и терпеливый репетитор сервиса «Мишка знает» для школьников.
Ученик: {name}, {grade} класс.
Предмет: {subject}.

Правила:
- Объясняй простым языком, без сложных терминов без необходимости
- Никогда не ругай и не осуждай ученика
- Если ученик ошибается — объясни почему, предложи попробовать снова
- Используй LaTeX для формул: $формула$ (inline) или $$формула$$ (блок)
- Будь кратким, но понятным. Разбивай на шаги
- Ты говоришь по-русски
```

### Промпт для генерации задания
```
Придумай одну задачу по предмету {subject} для ученика {grade} класса
по теме: {topic}.

Задача должна:
- Соответствовать уровню {grade} класса по программе РФ
- Иметь однозначный числовой или краткий текстовый ответ
- Быть решаемой за 5–10 минут

Формат ответа:
ЗАДАЧА: [текст задачи]
ОТВЕТ: [правильный ответ — только для системы, не показывай ученику]
```

### Промпт для проверки ответа
```
Задача: {task_text}
Правильный ответ: {correct_answer}
Ответ ученика: {user_answer}

Проверь ответ ученика. Если верно — похвали коротко.
Если неверно — объясни ошибку пошагово, покажи правильное решение.
Используй LaTeX для формул. Будь добрым и ободряющим.
```

## 8. Математическая клавиатура

Раскрывается кнопкой `∑` в поле ввода. Четыре вкладки:

| Вкладка | Символы |
|---|---|
| **Основные** | + − × ÷ = ≠ < > ≤ ≥ |
| **Функции** | sin cos tan cot arcsin arccos arctan √ |
| **Степени/дроби** | x² x³ xⁿ √x ∛x x/y |
| **Специальные** | ∫ ∑ ∞ π → ∈ ∉ ∩ ∪ Δ |

При нажатии кнопки — вставляет соответствующий LaTeX-код в поле ввода. Формулы рендерятся в пузыре сообщения через KaTeX.

## 10. API эндпоинты

| Метод | URL | Описание |
|---|---|---|
| POST | `/api/auth/register` | Регистрация |
| POST | `/api/auth/login` | Вход, возвращает JWT cookie |
| POST | `/api/auth/logout` | Выход |
| GET | `/api/chat/sessions` | Список сессий пользователя |
| POST | `/api/chat/sessions` | Создать новую сессию |
| GET | `/api/chat/sessions/[id]` | Сообщения сессии |
| POST | `/api/chat/stream` | **Стриминг ответа YandexGPT (SSE)** |
| POST | `/api/tasks/generate` | Генерация задачи ИИ |
| POST | `/api/tasks/check` | Проверка ответа ученика |
| GET | `/api/tasks/history` | История заданий пользователя |

### Стриминг (SSE)
```typescript
// app/api/chat/stream/route.ts
export async function POST(req: Request) {
  const { messages, subject, grade } = await req.json();

  const stream = new ReadableStream({
    async start(controller) {
      const response = await fetch(YANDEX_GPT_URL, {
        method: 'POST',
        headers: { Authorization: `Api-Key ${API_KEY}` },
        body: JSON.stringify({
          modelUri: `gpt://${FOLDER_ID}/yandexgpt`,
          completionOptions: { stream: true, maxTokens: 2000 },
          messages: [systemPrompt(subject, grade), ...messages],
        }),
      });

      for await (const chunk of response.body) {
        controller.enqueue(chunk);
      }
      controller.close();
    },
  });

  return new Response(stream, {
    headers: { 'Content-Type': 'text/event-stream' },
  });
}
```

## 11. Авторизация (без сторонних сервисов)

```typescript
// Регистрация
POST /api/auth/register
Body: { name, email, password, grade }
→ Хэшируем пароль: bcrypt.hash(password, 10)
→ Сохраняем в SQLite
→ Генерируем JWT: jose.sign({ userId, grade }, SECRET)
→ Ставим httpOnly cookie

// Вход
POST /api/auth/login
→ Находим пользователя по email
→ bcrypt.compare(password, hash)
→ Генерируем JWT, ставим cookie

// Middleware (middleware.ts)
→ Читаем cookie на каждом запросе к /app/*
→ Если нет — редирект на /login
```
